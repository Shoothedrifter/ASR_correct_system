# ASR 校对标注系统

基于 Vue 3 + TypeScript + Vite 构建的语音识别（ASR）校对与错误标注工具。支持实时录音、流式 ASR 识别、人工纠错及字符级差异分析。

---

## 功能特性

- **实时录音**：调用浏览器麦克风，将音频重采样至 16kHz/16-bit PCM 格式
- **流式 ASR**：通过 WebSocket 接入字节跳动豆包 ASR 服务，实时返回识别中间结果与最终结果
- **人工纠错**：识别完成后提供可编辑文本框，供标注员对 ASR 结果进行修改；文本框随内容自动扩展
- **校对小助手**：识别完成后实时分析用户的每一步编辑操作，并在面板底部显示识别准确率
- **字符级差异分析**：基于 LCS 算法自动计算原始识别文本与纠错文本之间的差异，可视化标注
- **三类错误分类**（统计）：
  - 错词 — 词语替换（ASR 识别错误），红色波浪下划线
  - 标点错误 — 标点符号替换，蓝色虚线下划线
  - 漏字 / 多字 — 仅在差异对比视图中高亮展示，不计入统计徽章
- **识别准确率**：基于标准 CER（字符错误率）公式实时计算，用户每次编辑后自动更新
- **安全凭证管理**：API 密钥通过 Vite 代理在服务端注入，不暴露于浏览器
- **离线 Mock 模式**：无需真实 ASR 服务即可进行本地开发与测试

## 快速开始

### 前置要求

- Node.js >= 18
- 字节跳动豆包 ASR 的 App Key 和 Access Key（或使用 Mock 模式）

### 安装依赖

```bash
npm install
```

### 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env`：

```env
VITE_DOUBAO_APP_KEY=your_app_key
VITE_DOUBAO_ACCESS_KEY=your_access_key
VITE_USE_MOCK=false
```

> 如需离线开发，将 `VITE_USE_MOCK` 设为 `true`，无需真实 ASR 凭证。

### 启动开发服务器

```bash
npm run dev
```

### 运行测试

```bash
npm test
```

### 生产构建

```bash
npm run build
```

## 校对小助手说明

识别完成后，左下方展示"校对小助手"面板，包含三个区域：

### 1. 实时编辑日志

用户每次修改识别文本，小助手即时记录操作，例如：

```
›  删除「我们」
›  将「，」改为「！」
›  添加「。」
```

操作检测采用**公共前缀/后缀差分**算法，精确定位变化区间后输出自然语言描述。

### 2. 差异对比视图

对比原始 ASR 文本与纠错后文本，四种标注样式：

| 类型 | 样式 | 计入统计 |
|------|------|------|
| 错词 | 红色波浪下划线 | ✓ |
| 标点错误 | 蓝色虚线下划线 | ✓ |
| 漏字（ASR 漏识别）| 橙色删除线 | — |
| 多字（ASR 多识别）| 绿色背景 | — |

### 3. 识别准确率

使用标准 **CER（字符错误率）** 公式：

```
准确率 = (1 − (替换数 + 删除数 + 插入数) / 参考文本长度) × 100%
```

- 参考文本 = 用户纠错后的文本（真值）
- 颜色区分：≥ 95% 绿色、≥ 80% 橙色、< 80% 红色
- 用户每次编辑后自动更新

## ASR 协议说明

与豆包 ASR 服务通信采用**二进制 WebSocket 帧**：

- 帧头：4 字节（含消息类型、序列标志、压缩类型等）
- 负载：gzip 压缩的 JSON 或 PCM 数据
- 序列号：正数为数据帧，负数为 EOF 帧，`0` 为配置帧
- 凭证通过 Vite 代理以 HTTP Header 注入，客户端代码中不含任何密钥

---

## 差异分析算法

`utils/diff.ts` 使用**最长公共子序列（LCS）**回溯算法：

1. 构建 ASR 原文与纠错文本的 LCS 矩阵
2. 回溯识别 Insert / Delete / Match 操作
3. 相邻的 Delete+Insert 对合并为**错词**（substitution）
4. 单独的 Delete 标记为**漏字**，单独的 Insert 标记为**多字**
5. 标点符号替换单独归类为**标点错误**
